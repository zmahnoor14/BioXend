What is BioXend?

Four work packages, one unified framework

Microbial biotransformation of xenobiotics — the process by which microorganisms chemically modify drugs, environmental contaminants, and other foreign compounds — is poorly represented in public databases due to a lack of reporting standards. BioXend fixes that.

WP1

Minimum Standards

Develop community-consensus minimum reporting standards (MIX-MB) covering chemicals, microbes, and biotransformation assays.

WP2

Metadata Automation

Automate metadata collection from user inputs, external APIs (ChEMBL, NCBI Taxonomy), and Bioschemas-aligned identifiers.

WP3

ChEMBL Submission

Nextflow DSL2 pipeline that generates six ChEMBL-ready submission files from simple tabular inputs — no manual formatting.

WP4

NFDI4Microbiota Workflow

Release BioXend as part of the NFDI4Microbiota workflow ecosystem for broad community adoption and long-term maintenance.

MIX-MB Standards

Minimum Information about Xenobiotics-Microbiome Biotransformation

Three interconnected sub-standards — each independently versioned — define exactly what metadata must be reported for reproducible and database-ready experiments. All standards are Bioschemas-aligned and follow FAIR principles.

MIX-MB(X)

Xenobiotics / Chemicals

Minimum metadata for the chemical substrate of biotransformation reactions — structure, identity, and classification.

  • CIDX Compound index identifier
  • SMILES Valid canonical SMILES or InChIKey
  • COMPOUND_NAME Preferred chemical name
  • ChEBI / ChEMBL ID Cross-reference identifiers
  • RIDX Linked reference index
MIX-MB(M)

Microbes / Organisms

Minimum metadata for the microbial agent — taxonomy, culture conditions, and sample provenance — aligned to Bioschemas Taxon profile.

  • NCBI TaxID Primary taxonomy identifier
  • ASSAY_ORGANISM Organism name (genus, species)
  • O2_CONDITION Oxygen requirement (controlled vocab)
  • SAMPLE_TYPE Bacterium, community, enzyme
  • ENA / BioSample ID Sequencing data links
MIX-MB(B)

Biotransformation / Assay

Minimum metadata for the experimental assay — conditions, analytical methods, products, and activity measurements.

  • AIDX Assay index identifier
  • ASSAY_TYPE bacteria / enzyme / community
  • ACTION_TYPE Substrate / Product / No Activity
  • TEXT_VALUE Measured activity value
  • ASSAY_DESCRIPTION Free-text assay description

Nextflow Pipeline

From spreadsheet to ChEMBL in one command

The BioXend Nextflow DSL2 pipeline reads two tabular input files and produces six ChEMBL-ready submission files automatically.

Inputs

  • smiles_list.txt  SMILES, Names, Study_ID
  • strain_cond.csv  strain, species, O2 condition
  • Reference parameters  DOI, title, authors

Outputs (results/)

  • REFERENCE.tsv
  • COMPOUND_RECORD.tsv
  • COMPOUND_CTAB.sdf
  • ASSAY.tsv & ASSAY_PARAM.tsv
  • ACTIVITY.tsv
1

GENERATE_REFERENCE R

Creates REFERENCE.tsv from DOI, title, authors, and year parameters.

2

GENERATE_COMPOUNDS Python + RDKit

Validates SMILES structures, queries ChEMBL API, and generates COMPOUND_RECORD.tsv and COMPOUND_CTAB.sdf.

3

GENERATE_ASSAY R

Builds one assay entry per strain/condition row in strain_cond.csv, outputting ASSAY.tsv and ASSAY_PARAM.tsv.

4

GENERATE_ACTIVITY R

Links compound CIDXs to assay AIDXs and constructs the ACTIVITY.tsv table with action types and measured values.

5

VALIDATE_SUBMISSION

Checks all six output files were generated and conform to ChEMBL submission requirements before deposition.

Quick Start

Start using BioXend

Choose the path that fits your workflow — download the template to prepare your data, or run the full Nextflow pipeline to generate ChEMBL submission files automatically.

STEP 01

Clone the repository

Get the latest version of BioXend from GitHub.

# Clone BioXend
git clone https://github.com/zmahnoor14/BioXend.git
cd BioXend
STEP 02

Set up environments

Create the Conda environments for Python and R dependencies.

# Python env (RDKit, pandas, requests)
conda env create -f envs/py_env.yml

# R env (tidyverse, optparse)
conda env create -f envs/r_env.yml
STEP 03

Run the pipeline

Execute the full Nextflow workflow with your input data.

nextflow run main.nf \
  --input_csv bin/smiles_list.txt \
  --strain_cond bin/strain_cond.csv \
  --outdir results \
  --doi "10.1000/example" \
  --title "Study Title" \
  --authors "Last, First" \
  --year 2025
